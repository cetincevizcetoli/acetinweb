YAPILACAK DÜZELTMELER:

1. Document root ve /public/ meselesi

En doğru canlı kurulum:
Domain document root doğrudan public/ klasörüne bakmalı.
Kullanıcı https://www.acetin.com.tr/ açtığında site açılmalı, URL’de /public/ görünmemeli.

DEPLOY_CHECKLIST.md içine bunu net yaz:
“Canlıda domain document root = public/ olmalı.”

Eğer hosting panelinde bu mümkün değilse fallback planı yaz:
- kök .htaccess ile public/ içine yönlendirme
- hassas klasörleri deny etme
- ama bunun ikinci tercih olduğunu açıkça belirt

Kodda sabit /public/ linkleri veya hatalı base path varsa kontrol et.
Sabit public/ bağımlılığı varsa düzelt.

2. config/local.php hijyeni

config/local.php gerçek ortam dosyasıdır.
Git’e ve paylaşılacak ZIP paketlerine girmemeli.

Yapılacaklar:
- config/local.php .gitignore içinde olmalı.
- Eğer Git tarafından takip ediliyorsa raporla ve git rm --cached önerisi ver.
- config/local.example.php oluştur veya güncelle.
- local.example.php içinde gerçek sunucu yolu, gerçek DB yolu, parola veya özel bilgi olmasın.
- Uygulama local.php yoksa anlaşılır hata versin veya example dosyasını işaret etsin.

Canlıda local.php elle oluşturulacak şekilde DEPLOY_CHECKLIST’e ekle.

3. SQLite ve backup dosyaları

SQLite DB htdocs/public dışında olmalı.

Beklenen canlı yapı örneği:
private/storage/fikrimvar.sqlite
private/storage/backups/

Yapılacaklar:
- Proje içinde *.sqlite, *.sqlite-*, *.bak, *.db dosyaları ara.
- Public altında veya repo içinde olmamaları gerektiğini raporla.
- .gitignore içinde bu türler kapalı olsun.
- Public sayfalarda DB yolu görünmesin.
- system-check sadece localde veya açıkça izinliyse çalışsın.

4. system-check.php

public/system-check.php canlıda bilgi sızdırmamalı.

Tercih edilen davranış:
- sadece localhost / 127.0.0.1 / ::1 için çalışsın
veya
- config/local.php içinde allow_system_check=true ise çalışsın
- aksi halde 403 veya sade “disabled” dönsün

Canlıda DB yolu, storage yolu, server path, PHP detayları göstermesin.

Yerelde çalışmaya devam etsin.

5. admin-local

admin-local yalnızca yerel kullanım içindir.

Yapılacaklar:
- localhost dışından erişim kesin reddedilmeli.
- Apache .htaccess + PHP kontrol birlikte çalışmalı.
- Login/footer içinde DB yolu, storage yolu veya local path görünmemeli.
- DEPLOY_CHECKLIST içine “admin-local canlıya yüklenmemeli; yüklenecekse IP/HTTPS ile ayrıca korunmalı” yaz.

6. deploy-manifest.json

deploy-manifest canlıda fazla bilgi sızdırmamalı.

Kontrol et:
- public/deploy-manifest.json herkese açık mı?
- içinde app/config/tools dosya listeleri var mı?
- DB hash’i veya DB boyutu var mı?
- sunucu yolu, local yol veya hassas dosya adı var mı?

Beklenen düzeltme:
- Manifest public olmak zorunda değilse public dışına taşı veya erişime kapat.
- Public kalması gerekiyorsa sadeleştir:
  sadece version, generated_at, public_asset_hash gibi zararsız bilgiler kalsın.
- DB hash’i, DB boyutu, app/config dosya listesi public manifest içinde olmasın.
- Deploy aracı local/admin tarafında detaylı manifest kullanabilir, ama public URL’de geveze manifest olmamalı.
